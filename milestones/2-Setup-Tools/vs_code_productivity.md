# Top 10 VSCode Shortcuts

## Navigation

* *Ctrl + P*: **Quick Open**. Search and open any file in your project instantly without clicking through the folder tree.
* *Ctrl + G*: **Go to Line**. Essential for jumping straight to a bug location reported in your terminal or logs.
* *Ctrl + Tab*: **Switch Tabs**. Quickly cycle through your open files, similar to switching apps on your OS.

## Editing

* *Ctrl + Alt + Down/Up*: **Multiple Cursors**. The "superpower" for editing multiple lines of code or variables simultaneously.
* *Alt + Up/Down*: **Move Line**. Move the current line of code up or down without cutting and pasting
* *Shift + Alt + Up/Down*: **Copy Line**. Quickly duplicate a line of code above or below—perfect for repetitive UI components.
* *Ctrl + D*: **Add Selection to Next Find Match**. Select the next occurrence of the current word, allowing for quick local renaming.

## Debugging

* *F5*: **Start Debugging**. Launch your application in debug mode to set breakpoints and inspect variables
* *Shift + F5*: **Stop Debugging**. Immediately kill the current debugging session and free up your ports

## Terminal Commands

* *Ctrl + `(Backtick)*: **Toggle Terminal**. Quickly hide or show the integrated terminal to run Git commands or check build status

## Reflection

The Multiple Cursors (Alt + Click) and Quick Open (Ctrl + P) will be the most impactful for my daily work at Focus Bear. Multiple cursors allow me to refactor repetitive code blocks or update multiple style properties in seconds, which is a common task in mobile UI development. Meanwhile, Quick Open is essential for navigating large codebases where searching through the file explorer manually would be too slow and disruptive to my "flow state."

Keyboard shortcuts significantly improve productivity by reducing "context switching" between the keyboard and the mouse, which keeps my hands in the home position and my mind on the logic. By automating repetitive navigation and editing tasks, I can execute ideas as fast as I think of them, effectively shortening the gap between thought and code.
