# Reflection

Kanban uses a visual board to make the "Work in Progress" (WIP) visible, which helps the team identify the bottlenecks and prevents any single developer from taking on too many tasks at once. By prioritizing the "Backlog" column, the team ensures that I am always pulling the most important mobile features to work on as soon as I have the capacity.

I can improve my workflow by strictly adhering to WIP limits, ensuring that I finish and test one feature before moving on to the next one to maintain high code quality. Additionally, I will be proactive in moving tasks to the "Blocked" column and adding comments the moment I hit a technical hurdle, which allows the team to help me resolve issues faster.
