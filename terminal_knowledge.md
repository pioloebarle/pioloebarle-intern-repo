**Reflection**

I chose to use the **Integrated Terminal in VS Code** using **Powershell**. I selected this terminal because it allows me to run Git commands and manage my project files without ever leaving my coding environment, which helps maintain my focus and workflow.

I customized my terminal by changing the theme to match my VS Code editor for better visibility and added aliases for common Git commands to speed up my development process. I also enabled the "split terminal" feature, which allows me to view my build logs and run shell commands side-by-side.

The most useful command I learned today was *git checkout -b [branch-name]*, as it allowed me to quickly create and switch to a new branch for my Pull Requests. I also found *ls -a* helpful for viewing hidden configuration files like *.gitignore* that are crucial for managing my repository.
