**Reflection**

I have used Git previously for various academic projects, such as working on the Course Management System, Mama's Recipe and my senior thesis project. In these contexts, Git was essential for collaborating with my teammates, as it allowed us to work on the same codebase without overwriting each other's contribution

I chose to use the built-in Git client in VS Code because it allows me to manage my version control directly within my coding environment. Having the source control side-bar right next to my Flutter or React Native files makes staging changes and resolving merge conflicts much faster and more intuitive.

The most interesting thing I learned today was how to properly resolve a merge conflict by manually combining logic from two different branches rather than just picking one over the other. I also gained a better understanding of why a Pull Request (PR) is a necessary gatekeeping step in professional workflows to ensure code quality before it reaches the main branch.
